# 浮点数格式详解

## 各格式结构对比

| 格式 | 符号位 | 指数位 | 尾数位 | 偏置 (bias) |
|-----|-------|-------|-------|------------|
| FP32 | 1 | 8 | 23 | 127 |
| FP16 | 1 | 5 | 10 | 15 |
| BF16 | 1 | 8 | 7 | 127 |
| FP8 E4M3 | 1 | 4 | 3 | 7 |
| FP8 E5M2 | 1 | 5 | 2 | 15 |

## 数值范围

| 格式 | 最小正正规数 | 最小正次正规数 | 最大值 |
|-----|-------------|---------------|-------|
| FP32 | 2^(-126) ≈ 1.18e-38 | 2^(-149) ≈ 1.4e-45 | ≈ 3.4e38 |
| FP16 | 2^(-14) ≈ 6.1e-5 | 2^(-24) ≈ 5.96e-8 | ≈ 65504 |
| BF16 | 2^(-126) ≈ 1.18e-38 | 2^(-133) ≈ 9.3e-41 | ≈ 3.4e38 |
| FP8 E4M3 | 2^(-6) ≈ 0.0156 | 2^(-9) ≈ 0.002 | ≈ 448 |
| FP8 E5M2 | 2^(-14) ≈ 6.1e-5 | 2^(-16) ≈ 1.5e-5 | ≈ 57344 |

## 正规数 vs 欑正规数

### 存储指数位 vs 实际指数

| 类型 | 存储指数位 E | 实际指数 | 尾数隐含值 |
|-----|-------------|---------|----------|
| 欑正规数 | E = 0 | 1 - bias | 0.xxx |
| 正规数 | E = 1 ~ max-1 | E - bias | 1.xxx |
| 特殊值 | E = max | — | ±inf / NaN |

### 计算公式

**正规数：**
```
值 = (-1)^sign × 1.xxx × 2^(E - bias)
```

**次正规数：**
```
值 = (-1)^sign × 0.xxx × 2^(1 - bias)
```

### 为什么次正规数用 1 - bias？

如果用 0 - bias，会产生 gap：

```
最小正规数（E=1）：  1.0 × 2^(1-bias)
最大次正规数（E=0，假设 0-bias）： 0.111...1 × 2^(0-bias) ≈ 1.0 × 2^(-bias)
差距：最小正规数 = 2 × 最大次正规数  ← 有空隙！
```

用 1 - bias，平滑衔接：

```
最小正规数（E=1）：  1.0 × 2^(1-bias)
最大次正规数（E=0，实际 1-bias）： 0.111...1 × 2^(1-bias) ≈ 1.0 × 2^(1-bias)
两者几乎相等，无缝衔接！
```

### 欑正规数的作用

| | 正规数 | 欑正规数 |
|---|---|---|
| 精度 | 固定 | 逐渐丢失（越靠近0精度越低） |
| 范围 | 2^(emin) ~ 2^(emax) | 0 ~ 2^(emin) |
| 用途 | 正常计算 | 平滑下溢到0，避免突然变成0 |

次正规数实现"渐进下溢"（gradual underflow），填补 0 和最小正规数之间的空隙。

## Python 验证

```python
import torch

# FP32
print(torch.finfo(torch.float32).smallest_normal)      # 1.175e-38
print(torch.finfo(torch.float32).smallest_subnormal)   # 1.401e-45

# FP16
print(torch.finfo(torch.float16).smallest_normal)      # 6.104e-05
print(torch.finfo(torch.float16).smallest_subnormal)   # 5.96e-08

# BF16
print(torch.finfo(torch.bfloat16).smallest_normal)     # 1.175e-38
print(torch.finfo(torch.bfloat16).smallest_subnormal)  # 9.18e-41
```

## FP8 两种格式

**E4M3：**
- 更大精度范围（尾数 3 位）
- 更小动态范围（指数 4 位）
- 适合激活值存储

**E5M2：**
- 更大动态范围（指数 5 位）
- 更低精度（尾数 2 位）
- 适合梯度存储（防止溢出）

FP8 是 AI 训练加速的新标准， NVIDIA H100 / AMD MI300 等硬件支持。
